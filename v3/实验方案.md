## 1. 项目定位

### 1.1 核心问题

> 不同后训练方法（SFT / RFT / Online RFT / DPO / GRPO）如何重塑小型语言模型在数学推理任务上的**思维链长度**和**答案分布**？这些行为变化是反映了模型**真实能力的扩展**，还是仅仅是**采样分布的锐化**？

这一核心问题分解为两条主线：

- **主线 A：思维链长度演化**——后训练方法如何改变模型的"思考长度"？长度变化与正确率提升、与题目难度之间的关系是什么？
- **主线 B：答案分布演化**——后训练方法是在做"分布锐化（sharpening）"（让本来会的题更稳定地答对），还是"能力扩展（capability extension）"（让本来不会的题学会做），还是"模式坍缩（mode collapse）"（牺牲多样性换取尖锐性）？

### 1.2 项目立足点

本项目以 **DeepSeekMath (Shao et al., 2024) Figure 5** 为出发点。原论文在 DeepSeekMath-Instruct 1.3B 上对比了 RFT / Online RFT / GRPO，但属于探索性实验，仅报告 accuracy 曲线，未深入分析行为变化机制。本项目沿三个维度补充：

1. **方法学严谨性**：token-aligned 算力对齐、instance-level 数据隔离、单模型受控对比
2. **算法覆盖面**：在原论文基础上加入 SFT 和 DPO，形成跨 SL / 离线 RL / 在线 RL 三类的完整对比
3. **聚焦的行为分析**：不止报告"涨了多少 acc"，而是诊断"长度怎么变、分布怎么变、变化模式属于哪一类"

### 1.3 项目边界

| 是 | 不是 |
|----|------|
| 已有方法的受控验证 + 行为分析 | 提出新算法 |
| 单模型受控对比（Gemma-2-2B-IT） | 跨规模 / 跨模型家族研究 |
| 聚焦于 CoT 长度 + 答案分布 | 全方位机制分析（错误分类、logit 诊断等列为展望） |
| 工业相关（IT 起点） | base 模型起点的研究（属 R1-Zero 范畴） |

### 1.4 Scope 声明

仅研究单一模型、IT 起点。机制分析仅聚焦于**思维链长度**和**答案分布**两个维度。其他诊断维度（错误类型分类、logit 级 KL 分析、attention 模式、跨模型验证等）**主动声明 out of scope**，列为 Section 10 研究展望。

**为什么聚焦**：
- 这两个维度直接对应当前 LLM reasoning 研究最热的 narrative（test-time compute scaling、sharpening vs extension）
- 聚焦让项目从"对比研究"升级为"诊断研究"，叙事更连贯
- 主动 declare scope 是 senior researcher 的习惯——本科项目能聚焦反而显得思路成熟

## 2. 实验设计

### 2.1 模型

- **Base model**：`google/gemma-2-2b-it`
- 全部 6 组实验共享相同起始权重，确保直接可比

### 2.2 数据

GSM8K train（共 7473 题）切分：

| 切片 | 数量 | 用途 |
|------|------|------|
| `D_rl` | ~6973 | RFT 采样池、Online RFT prompt、DPO pair 来源、GRPO prompt |
| `D_dev` | 500 | 超参选择（DPO β、GRPO KL 系数、learning rate） |

**评估**：
- **In-domain**：GSM8K test（1319 题）
- **OOD**：MATH test（5000 题，因算力原因抽样 500 题评估）

**数据隔离原则**：
- 所有 RL 类方法（RFT、Online RFT、GRPO）共享同一 `D_rl` prompt 池——保证方法间公平
- `D_dev` 永不用于评估——严格防止 test set 污染
- SFT (E2) 也在 `D_rl` 上训练，与 RL 方法保持训练 prompt 一致（被对比的是算法，不是数据）

### 2.3 算力对齐

**主轴：tokens-seen**。每个方法的主实验 budget 为 **~50M training tokens**。

| 方法 | token 计算方式 |
|------|---------------|
| SFT | batch × seq_len × steps |
| RFT | batch × seq_len × steps（filter 之后） |
| DPO | batch × seq_len × 2（chosen + rejected）× steps |
| Online RFT | rollout × seq_len × steps |
| GRPO | rollout × seq_len × group_size × steps |

**为什么用 tokens-seen，不用 wall-clock 或 epoch**：
- Wall-clock 跨硬件不可复现
- Epoch 跨方法不可比（RL "epoch" ≠ SFT "epoch"）
- Tokens-seen = 信息暴露量，是最干净的受控变量

### 2.4 实验列表（共 6 组）

| ID | 方法 | 起点 | 训练数据 | 目的 |
|----|------|------|---------|------|
| E1 | zero-shot | Gemma-2-2B-IT | — | Baseline（无训练状态作为参照） |
| E2 | SFT | Gemma-2-2B-IT | `D_rl`（ground-truth CoT） | 验证灾难性遗忘假设 |
| E3 | RFT | Gemma-2-2B-IT | 从 `D_rl` 采样 + filter | 离线 self-improvement |
| E4 | Online RFT | Gemma-2-2B-IT | 在 `D_rl` 上 on-policy 采样 | 在线 self-improvement |
| E5 | DPO | Gemma-2-2B-IT | 从 `D_rl` 构造的 preference pair | 离线偏好学习 |
| E6 | GRPO | Gemma-2-2B-IT | 在 `D_rl` 上 on-policy rollout | Group-relative RL |

### 2.5 评估协议

每个方法的每个 checkpoint 都跑：

| 指标 | 采样方式 | 反映什么 |
|------|---------|---------|
| pass@1 | T=0, greedy | Sharpness（尖锐性） |
| pass@8 | T=0.7, n=8 | 分布覆盖 |
| pass@32 | T=0.7, n=32 | 能力上限 |
| maj@8 | T=0.7, n=8, 多数投票 | Self-consistency |

在 **GSM8K test（in-domain）** 和 **MATH test 500 子集（OOD）** 上各算一遍。

**Checkpointing**：每个 run 均匀保存 10–15 个 checkpoint，用于画 learning curve。

## 3. 行为分析框架（项目核心）

本项目的**真正深度**在这一节。我们不止报告 accuracy，而是诊断后训练方法如何重塑模型的 reasoning 行为。

**应用范围**：对全部 5 个方法（SFT、RFT、Online RFT、DPO、GRPO）做主线 A、B 的核心分析。原计划只对 3 个核心方法做的限制被取消——因为聚焦后总工作量下降，全方法分析变得可行。

**优先级标注**：
- 🔴 **必做**：项目的核心叙事支撑，缺了故事不完整
- 🟡 **选做**：时间允许就做，做了显著加分

**统一 framing**（简历/面试可直接用）：

> 我们将后训练方法的效果分解为两个独立维度：**思维链长度的演化**（模型"想多久"）和**答案分布的演化**（模型"想得多准/多稳"）。通过这两个维度的交叉分析，我们诊断各方法究竟是在做"分布锐化"、"能力扩展"还是"模式坍缩"，而不仅仅报告 accuracy 提升。

---

## 主线 A：思维链长度演化

### 3.A.1 🔴 长度分布

**测什么**：每个方法的 final checkpoint 在 GSM8K test 1000 题上的 response 长度分布。

**指标**：
- 平均长度（token 数）
- 中位数、25/75/95 分位数
- 长度直方图（与 baseline 叠加对比）

**回答什么问题**：
- 不同方法训练后，模型变得更"啰嗦"还是更"简洁"？
- 长度分布形态变了吗？（从单峰变双峰？方差变大？）

---

### 3.A.2 🔴 长度训练动态

**测什么**：每个方法的 10–15 个 checkpoint 上的平均长度曲线。

**画图**：X 轴是训练 step（或 tokens-seen），Y 轴是平均 response 长度。

**回答什么问题**：
- 长度是单调上升还是震荡？
- 哪些方法导致长度爆炸性增长？（GRPO 是著名的"length hacking"高发方法）
- 长度增长在什么时候停止/反转？

**预期 finding**（待验证）：

> GRPO 训练过程中 CoT 长度持续增长（从 80 token 增至 220 token），且无饱和迹象；Online RFT 出现温和增长（80 → 130 token）；SFT 几乎不变（80 → 85 token）。这呼应 R1 论文观察到的"long CoT 涌现是 RL 的副产物"现象。

---

### 3.A.3 🔴 长度 vs 正确率

**测什么**：将每个方法的 response 按长度分桶（如每 50 token 一桶），看每桶的正确率。

**画图**：X 轴是长度桶，Y 轴是该桶内的 accuracy，每个方法一条曲线。

**回答什么问题**：
- 长 = 更对吗？还是有 sweet spot？
- 不同方法的"长度-正确率"曲线形态是否一致？
- 是否有方法呈现倒 U 形（过长反而错）？

**预期 finding**（待验证）：

> 所有方法都呈现倒 U 形——超过某个长度后正确率下降。但 GRPO 的 sweet spot 显著长于 SFT（180 vs 100 token），说明 RL 让模型有效利用更长的推理空间。

---

### 3.A.4 🔴 长度 vs 难度

**测什么**：在难度切片（Easy/Medium/Hard，按 base pass@1 划分）上分别看长度。

**回答什么问题**：
- 模型是否对难题"想得更久"？
- 哪些方法学会了"按难度调整长度"？（这是 high-level reasoning 的标志）
- 哪些方法对所有难度都用同样长度？（说明只学了"格式"，没学"自适应"）

**预期 finding**（待验证）：

> GRPO 在 Hard 题上长度比 Easy 题长 80%，SFT 仅长 15%——RL 让模型学会了"难题想久一点"的元能力。

---

### 3.A.5 🟡 Token 数 vs Step 数

**测什么**：分别统计 token 长度和"逻辑步骤数"（按 `=` 或换行计数）。

**为什么需要这个**：长度变长可以是因为：
- (a) 多想了几个推理步骤（好的长）
- (b) 每步话更多了（中性的长）
- (c) 重复 / repetition hacking（坏的长）

只看 token 数无法区分这三种。同时看 step count，能看出**"是真的多想了，还是只是话多了"**。

**为什么是 🟡 选做**：增加少量统计代码，**强烈推荐做**，因为它直接捍卫 finding A.2 的结论强度。如果 GRPO 长度涨 2 倍但 step 数只涨 20%，那"长 CoT 涌现"的解读就要打折。

## 主线 B：答案分布演化

这一线回答你最在意的问题：**"是真能力变强，还是只是分布变了？"**

### 3.B.1 🔴 pass@k / maj@k 联合曲线分析（项目灵魂）

**这是整个项目最重要的单一分析**。直接回答"是真能力变强还是分布变了"，对标 DeepSeekMath Figure 7。

#### 测什么

100 题子集上对每个方法采样 32 次（T=0.7），从这一组采样里同时算出 **maj@k 和 pass@k 在多个 k 上的值**：

- k 网格：**1, 2, 4, 8, 16, 32**
- 对每个 k 同时算 maj@k 和 pass@k
- 用 unbiased estimator 从 32 次采样里计算各 k 值，零额外采样

#### 主图（核心 deliverable）

**结构对标 DeepSeekMath Figure 7**：

- **左图**：GSM8K 上的 maj@k vs pass@k 联合曲线
- **右图**：MATH 上的 maj@k vs pass@k 联合曲线
- **每张图 10 条曲线**：5 个方法 × 2 类指标（maj@k 实线，pass@k 虚线）
- X 轴：k（log scale，1 到 32）
- Y 轴：accuracy

#### 以 k 为变量能探查的 6 层信号

每个方法的曲线形状本身就是诊断信号：

| 曲线特征 | 揭示什么 |
|---------|---------|
| **起点（k=1）** | pass@1，模型默认输出质量 |
| **小 k 斜率（k=1→8）** | confidence 离散度：陡 = 多次采样能 boost、平 = 模型自信稳定 |
| **饱和速度** | mode collapse 信号：k=4 就饱和说明采样多样性极低 |
| **饱和值（k=32）** | 多数决能力上限 |
| **maj@k 与 pass@k 差距** | confidence 与 correctness 的对齐程度（差距大 = 能碰对但 maj 跟不上）|
| **整体形态分类** | 把方法分成"健康高自信 / 早期饱和 / 持续受益"等 pattern |

#### 诊断框架（核心论断）

DeepSeekMath Figure 7 提出的论断：**"RL enhances Maj@K but not Pass@K"**——RL 在做的不是教模型新能力，而是把模型已有的概率信号集中到正确答案上。

我们将这一论断在 5 种方法 + 2 种 setting（in-domain / OOD）上验证或反驳。可能的发现路径：

| 观察现象 | 诊断结论 |
|---------|---------|
| RL 后 maj@k 整体上移、pass@k 不变 | 验证 DeepSeekMath 论断（**Sharpening / Confidence alignment**）|
| RL 后 maj@k 和 pass@k 都上移 | 反驳 DeepSeekMath 论断（**真 Capability extension**——可能因为 2B 模型尚未触及能力上限）|
| RL 后 pass@k 上但 maj@k 不上 | 罕见现象（模型变得更会"碰对"但 confidence 没对齐）|
| 在 GSM8K 上 RL 提升 maj@k，在 MATH OOD 上不提升 | RL 学到的"自信"是 in-domain 现象，不泛化 |

#### 预期 finding（待验证）

候选叙事 A（验证 DeepSeekMath）：

> "在 Gemma-2-2B-IT 上我们复现并扩展了 DeepSeekMath Figure 7 的核心观察：GRPO、Online RFT 显著提升 maj@k 整条曲线（GSM8K 上 maj@16 从 X% 涨到 Y%），但 pass@k 曲线几乎不变——RL 在做 confidence alignment 而非 capability extension。这一结论在 5 种方法上一致，且独立于模型规模（2B vs DeepSeekMath 7B）。"

候选叙事 B（部分反驳）：

> "与 DeepSeekMath 的 7B 模型不同，2B 模型上 GRPO 同时提升 maj@k 和 pass@k（pass@32 涨 X 点），表明在弱模型尚未触及能力上限时，RL 仍能扩展模型的能力包络。'RL only sharpens' 这一论断的边界条件是模型规模 / 能力饱和度。"

候选叙事 C（OOD 差异）：

> "GRPO 在 GSM8K 上 maj@k 显著上移、pass@k 不变（DeepSeekMath 模式）；但在 MATH OOD 上 maj@k 几乎不变——RL 学到的 confidence calibration 是 in-domain 过拟合，没有迁移到分布外任务。"

**任何一种叙事都是好 finding**——A 是 replication study 的胜利，B 是新发现，C 是泛化边界。三者中任何一个都能撑起项目核心论点。

#### 工作量

100 题 × 32 次采样 = 0.8M token inference，vLLM 在 L4 上约 10 分钟搞定。**所有 maj@k / pass@k 在不同 k 上的值都从这一次采样算出来**——不需要额外采样。

### 3.B.2 🔴 Solution diversity

**测什么**：每题 8 次采样里有多少种"不同的解法"。

**简化定义**：两个 response 算同一种解法 ⟺ final answer 相同 **且** 推理 step 数差 ≤ 1。

**回答什么问题**：
- 训练后模型的"解法多样性"是上升还是下降？
- 是否存在"acc 涨但 diversity 跌"的 trade-off？（mode collapse 的 signature）

**预期 finding**（待验证）：

> SFT 后 solution diversity 几乎不变（4.2 → 4.1）——它没改变模型的探索行为。GRPO 后 diversity 从 4.2 降到 2.8——RL 显著压缩了探索空间。这是 RL 的代价，也是 mode collapse 的典型 signature。

---

### 3.B.3 🔴 Answer distribution entropy

**测什么**：每题 8 次采样里，**final answer 的熵**。

**计算方式**：把 8 个 final answer 当成一个分布，算 Shannon entropy。
- entropy = 0：8 次都给同一个答案（极度自信，可能是 mode collapse）
- entropy = log(8)：8 次都不一样（极度不确定）

**为什么这个指标重要**：直接量化"答案分布有多锐利"。比 solution diversity 更精确，因为它不依赖"解法相同"的人为定义。

**画图**：每个方法的 final ckpt 上，对所有题计算 entropy，画分布。

**预期 finding**（待验证）：

> SFT 训练后 answer entropy 中位数从 0.8 → 0.6——温和锐化。GRPO 训练后从 0.8 → 0.2——剧烈锐化，大量题目的 8 次采样给同一个答案。

---

### 3.B.4 🔴 In-domain vs OOD 对比

**测什么**：所有 B.1-B.3 指标都在 GSM8K test 和 MATH test 上各算一遍，对比。

**回答什么问题**：
- RL 学到的"分布特征"（锐化、低 diversity、maj@k 提升）是否泛化到 OOD？
- 是否存在"in-domain 锐化 + OOD 仍模糊"的现象？
- DeepSeekMath Figure 7 的"RL only enhances Maj@K"现象在 OOD 上是否仍然成立？

**预期 finding**（待验证）：

> GRPO 在 GSM8K 上 entropy 显著降低、maj@k 整条曲线上移；但在 MATH OOD 上 entropy 几乎不变、maj@k 曲线接近 baseline——RL 学到的 confidence calibration 是 in-domain 现象，没有迁移到 OOD。这与 "RL 过拟合 reward distribution" 假设一致，也是对 DeepSeekMath Figure 7 的 OOD 边界补充。

---

### 3.B.5 🟡 错答分布坍缩

**测什么**：错答里有多少种不同的"错误答案"。

**为什么有趣**：
- 训练前：模型错的题里，可能给出 5 种不同错答（错得各种各样）
- GRPO 后：模型错的题里，可能 8 次都给同一个错答（自信地错）

**回答什么问题**：训练除了让"对的题更对"，是否也让"错的题更错"？

**为什么是 🟡 选做**：是 B.2/B.3 的延伸，故事很漂亮但优先级稍低。

## 8. Story 大纲（用于简历 + 面试）

**One-liner**：
> 在 Gemma-2-2B-IT 上对 5 种后训练方法做受控对比，从**思维链长度**和**答案分布**两个维度诊断各方法的行为塑造模式——区分"真能力扩展"、"分布锐化"和"模式坍缩"，而非仅报告 accuracy 提升。

**Findings（实验跑完后回填，全部围绕长度 + 分布两条主线）**：

1. **[长度演化]** RL 类方法（GRPO、Online RFT）显著拉长 CoT（GRPO: 80 → 220 token），SFT 几乎不变；GRPO 在 Hard 题上长度比 Easy 题长 X%，体现"按难度调整长度"的元能力。

2. **[长度-正确率关系]** 所有方法都呈现长度-正确率倒 U 形，但 GRPO 的 sweet spot 显著长于 SFT（180 vs 100 token）——RL 让模型有效利用更长的推理空间。

3. **[分布诊断 - 项目核心]** 通过 **maj@k 与 pass@k 在不同 k 上的联合曲线分析**（对标 DeepSeekMath Figure 7），三种行为模式被区分：
   - SFT / DPO：maj@k 整条曲线上移、pass@k 不变（纯 sharpening）
   - GRPO：maj@k 整条曲线显著上移、pass@k 同时上移（capability extension），但伴随 mode collapse signature（曲线饱和速度加快、entropy 降低）
   - Online RFT：介于两者之间

4. **[OOD 泛化]** RL 学到的"自信"（answer entropy 降低）是 in-domain 的，在 MATH OOD 上消失——RL 可能过拟合到 in-domain reward distribution。

5. **[遗忘]** IT + SFT 出现灾难性遗忘（GSM8K acc 下降 X 点 + CoT 长度反常缩短），与 R1-Zero 路线"小模型 IT 起点应跳过 SFT"的假设对齐。

**简历 bullet（目标版本）**：
> 在 Gemma-2-2B-IT 上设计并执行了 5 种后训练方法的受控对比实验，聚焦于思维链长度和答案分布两个行为维度。通过 maj@k 与 pass@k 在多个 k 值上的联合曲线分析（对标 DeepSeekMath Figure 7），区分了 SFT 类方法的"confidence alignment"与 GRPO 的"capability extension + mode collapse trade-off"，并发现 IT + SFT 引发灾难性遗忘（GSM8K 下降 12 点），支持"小模型 IT 起点应直接 RL"的工程实践。

## 10. 研究展望（Future Work）

本项目主动 declare scope 在"思维链长度 + 答案分布"两个维度。下列方向**确认有研究价值但本项目不实施**，列出以展示我们对完整诊断空间的认识，也为后续工作提供路线图。

### 10.1 错误类型分类

**目标**：用 LLM-as-judge 对错答做 6 类分类（setup error / computation error / logical gap / format error / truncation / repetition），观察各方法消除/引入错误类型的差异。

**预期价值**：识别 RL 训练的标志性副作用（如 GRPO 后 repetition error 上升 = reward hacking 信号）。

**未实施原因**：与项目核心 narrative（长度 + 分布）相关性弱；LLM judge 需要 sanity check 工程。

---

### 10.2 Logit / KL 级诊断

**目标**：

| 指标 | 揭示什么 |
|------|---------|
| NLL on correct CoT | 模型对正确解的"愿意"程度 |
| Top-1 prob at decision tokens | 关键 token（数字、运算符）的局部 confidence |
| Entropy across timesteps | 推理过程中模型在哪些位置"犹豫" |
| KL(π_after \|\| π_base) per token | 后训练改了模型的"哪些位置" |

**预期价值**：能区分"SFT 主要改格式 token、GRPO 主要改推理 step"这类 fine-grained locality 现象。

**未实施原因**：实现复杂（需在 inference 时同时持有 base 和 trained model 的 logits）；与 pass@k 解构有部分重叠；解释门槛高。

---

### 10.3 Attention 模式分析

**目标**：观察训练前后 attention pattern 在数学推理 token 上的变化。

**潜在发现**：RL 可能让模型对"问题中的关键数字"分配更多 attention；SFT 可能不改变 attention 分布只改变 output projection。

**未实施原因**：mechanistic interpretability 是独立子领域，工作量超出本项目预算。

---

### 10.4 跨模型 / 跨规模验证

**目标**：在 Qwen2.5-1.5B、Llama-3.2-1B 等其他小模型上重复主实验，验证"长度 + 分布"行为模式的普遍性。

**预期价值**：当前结论限于 Gemma-2-2B，跨模型验证后可声明"小模型上 X 现象普遍存在"。

**未实施原因**：算力 ×3，时间不允许；属于"项目工业化"而非"项目深化"。

---

### 10.5 与 process reward (PRM) 的对比

**目标**：当前 GRPO 实现只用 outcome supervision，加入 process supervision (PRM) 后，CoT 长度和分布特征是否有定性差异？

**预期价值**：DeepSeekMath Figure 5 里 GRPO+PS 比 GRPO+OS 强约 1 点，但行为差异未被分析。

**未实施原因**：需要训练 / 借用 PRM，数据和算力双重负担。

---

### 10.6 长 CoT 的扩展性

**目标**：当前主实验 max_response_length 设为 512 token，限制了"long CoT 涌现"现象的上限观察。在更大 max length（如 2048）下，GRPO 是否会展现 R1 论文中的"think for thousands of tokens"现象？

**预期价值**：在 2B 量级模型上验证"测试时计算扩展（test-time compute scaling）"的可行边界。

**未实施原因**：max length 翻 4 倍 = 单步算力翻 4 倍，预算无法承担；属于"scale-up 研究"而非"scale-down 验证"。

---

- 简历 / SOP 上提到"future work includes ..."一句，让评审看到你想得周全
- 面试时被问"为什么不做 X"时直接 reference Section 10，不会被打个措手不及
- 这本身就是一份"如果项目继续做，能往哪些方向走"的研究路线图——对申请研究生时也有用
